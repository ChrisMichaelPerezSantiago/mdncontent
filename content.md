## ✔️ content 
 ---<br/>title: content<br/>slug: Web/CSS/content<br/>tags:<br/>  - CSS<br/>  - CSS Counter<br/>  - CSS Property<br/>  - Generated Content<br/>  - Reference<br/>  - recipe:css-property<br/>browser-compat: css.properties.content<br/>---<br/>{{CSSRef}}<br/><br/>The **`content`** [CSS](/en-US/docs/Web/CSS) property replaces an element with a generated value. Objects inserted using the `content` property are **anonymous [replaced elements](/en-US/docs/Web/CSS/Replaced_element)**.<br/><br/>```css<br/>/* Keywords that cannot be combined with other values */<br/>content: normal;<br/>content: none;<br/><br/>/* <image> values */<br/>content: url(http://www.example.com/test.png);<br/>content: linear-gradient(#e66465, #9198e5);<br/>content: image-set(image1x.png 1x, image2x.png 2x);<br/><br/>/* alt text for generated content, added in the Level 3 specification */<br/>content: url(http://www.example.com/test.png) / This is the alt text;<br/><br/>/* <string> value */<br/>content: prefix;<br/><br/>/* <counter> values, optionally with <list-style-type> */<br/>content: counter(chapter_counter);<br/>content: counter(chapter_counter, upper-roman);<br/>content: counters(section_counter, .);<br/>content: counters(section_counter, ., decimal-leading-zero);<br/><br/>/* attr() value linked to the HTML attribute value */<br/>content: attr(value string);<br/><br/>/* Language- and position-dependent keywords */<br/>content: open-quote;<br/>content: close-quote;<br/>content: no-open-quote;<br/>content: no-close-quote;<br/><br/>/* Except for normal and none, several values can be used simultaneously */<br/>content: open-quote counter(chapter_counter);<br/><br/>/* Global values */<br/>content: inherit;<br/>content: initial;<br/>content: revert;<br/>content: revert-layer;<br/>content: unset;<br/>```<br/><br/>## Syntax<br/><br/>### Values<br/><br/>- `none`<br/>  - : When applied to a pseudo-element, the pseudo-element is not generated. If applied to an element, the value has no effect.<br/>- `normal`<br/>  - : Computes to `none` for the `::before` and `::after` pseudo-elements.<br/>- {{cssxref(&lt;string&gt;)}}<br/>  - : Specifies the alt text for the element. This value can be any number of text characters. Non-Latin characters must be encoded using their Unicode escape sequences: for example, `\000A9` represents the copyright symbol.<br/>- {{cssxref(&lt;image&gt;)}}<br/>  - : An {{cssxref(&lt;image&gt;)}}, denoted by the {{cssxref(url, url())}} or {{cssxref(&lt;gradient&gt;)}} data type, or part of the webpage, defined by the {{cssxref(element, element())}} function, denoting the content to display.<br/>- {{cssxref(counter, counter())}}<br/><br/>  - : The value of a [CSS counter](/en-US/docs/Web/CSS/CSS_Counter_Styles/Using_CSS_counters), generally a number produced by computations defined by {{cssxref(&lt;counter-reset&gt;)}} and {{cssxref(&lt;counter-increment&gt;)}} properties. It can be displayed using either the {{cssxref(counter, counter())}} or {{cssxref(counters, counters())}} function.<br/><br/>    The {{cssxref(counter, counter())}} function has two forms: 'counter(_name_)' or 'counter(_name_, style)'. The generated text is the value of the innermost counter of the given name in scope at the given pseudo-element. It is formatted in the specified {{cssxref(&lt;list-style-type&gt;)}} (`decimal` by default).<br/><br/>    The {{cssxref(counters, counters())}} function also has two forms: 'counters(_name_, _string_)' or 'counters(_name_, _string_, _style_)'. The generated text is the value of all counters with the given name in scope at the given pseudo-element, from outermost to innermost, separated by the specified string. The counters are rendered in the indicated {{cssxref(&lt;list-style-type&gt;)}} (`decimal` by default).<br/><br/>- `attr(x)`<br/>  - : The value of the element's attribute `x` as a string. If there is no attribute `x`, an empty string is returned. The case-sensitivity of attribute names depends on the document language.<br/>- `open-quote` | `close-quote`<br/>  - : These values are replaced by the appropriate string from the {{cssxref(quotes)}} property.<br/>- `no-open-quote` | `no-close-quote`<br/>  - : Introduces no content, but increments (decrements) the level of nesting for quotes.<br/><br/>## Accessibility concerns<br/><br/>CSS-generated content is not included in the [DOM](/en-US/docs/Web/API/Document_Object_Model/Introduction). Because of this, it will not be represented in the [accessibility tree](/en-US/docs/Learn/Accessibility/What_is_accessibility#accessibility_apis) and certain assistive technology/browser combinations will not announce it. If the content conveys information that is critical to understanding the page's purpose, it is better to include it in the main document.<br/><br/>- [Accessibility support for CSS generated content – Tink](https://tink.uk/accessibility-support-for-css-generated-content/)<br/>- [Explanation of WCAG, Guideline 1.3 – MDN](/en-US/docs/Web/Accessibility/Understanding_WCAG/Perceivable#guideline_1.3_%e2%80%94_create_content_that_can_be_presented_in_different_ways)<br/>- [Understanding Success Criterion 1.3.1 | W3C Understanding WCAG 2.0](https://www.w3.org/TR/UNDERSTANDING-WCAG20/content-structure-separation-programmatic.html)<br/><br/>## Formal definition<br/><br/>{{cssinfo}}<br/><br/>## Formal syntax<br/><br/>{{csssyntax}}<br/><br/>## Examples<br/><br/>### Headings and quotes<br/><br/>This example inserts quotation marks around quotes, and adds the word Chapter before headings.<br/><br/>#### HTML<br/><br/>```html<br/><h1>5</h1><br/><p>According to Sir Tim Berners-Lee,<br/>  <q cite=http://www.w3.org/People/Berners-Lee/FAQ.html#Internet>I was<br/>    lucky enough to invent the Web at the time when the Internet<br/>    already existed - and had for a decade and a half.</q><br/>  We must understand that there is nothing fundamentally wrong<br/>  with building on the contributions of others.<br/></p><br/><br/><h1>6</h1><br/><p>According to the Mozilla Manifesto,<br/>  <q cite=http://www.mozilla.org/en-US/about/manifesto/>Individuals<br/>    must have the ability to shape the Internet and<br/>    their own experiences on the Internet.</q><br/>  Therefore, we can infer that contributing to the open web<br/>  can protect our own individual experiences on it.<br/></p><br/>```<br/><br/>#### CSS<br/><br/>```css<br/>q {<br/>  color: blue;<br/>}<br/><br/>q::before {<br/>  content: open-quote;<br/>}<br/><br/>q::after {<br/>  content: close-quote;<br/>}<br/><br/>h1::before  {<br/>  content: Chapter ;  /* The trailing space creates separation<br/>                           between the added content and the<br/>                           rest of the content */<br/>}<br/>```<br/><br/>#### Result<br/><br/>{{EmbedLiveSample('Headings_and_quotes', '100%', 200)}}<br/><br/>### Image combined with text<br/><br/>This example inserts an image before the link. Content added with `::before` or `::after` will not be generated as the contents of the element have been replaced.<br/><br/>#### HTML<br/><br/>```html<br/><div id=replaced>Mozilla</div><br/>```<br/><br/>#### CSS<br/><br/>```css<br/>#replaced {<br/>  content: url(mdn.svg);<br/>}<br/><br/>#replaced::after { /* will not show if element replacement is supported */<br/>  content:  ( attr(id) );<br/>}<br/>```<br/><br/>#### Result<br/><br/>{{EmbedLiveSample('Element_replacement', '100%', 200)}}<br/><br/>## Specifications<br/><br/>{{Specifications}}<br/><br/>## Browser compatibility<br/><br/>{{Compat}}<br/><br/>## See also<br/><br/>- [Replaced elements](/en-US/docs/Web/CSS/Replaced_element)<br/>- {{Cssxref(::after)}}<br/>- {{Cssxref(::before)}}<br/>- {{Cssxref(::marker)}}<br/>- {{Cssxref(quotes)}}<br/>- {{cssxref(url, url())}} function<br/>
