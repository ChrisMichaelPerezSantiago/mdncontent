## ✔️ blend-mode 
 ---<br/>title: <blend-mode><br/>slug: Web/CSS/blend-mode<br/>tags:<br/>  - Blend modes<br/>  - CSS<br/>  - CSS Data Type<br/>  - Compositing<br/>  - Compositing and Blending<br/>  - Data Type<br/>  - Reference<br/>  - color<br/>browser-compat: css.types.blend-mode<br/>---<br/>{{CSSRef}}<br/><br/>The **`<blend-mode>`** [CSS](/en-US/docs/Web/CSS) [data type](/en-US/docs/Web/CSS/CSS_Types) describes how colors should appear when elements overlap. It is used in the {{cssxref(background-blend-mode)}} and {{cssxref(mix-blend-mode)}} properties.<br/><br/>## Syntax<br/><br/>The `<blend-mode>` data type is defined using a keyword value chosen from the list below.<br/><br/>### Values<br/><br/>- `normal`<br/>  - : The final color is the top color, regardless of what the bottom color is.<br/>    The effect is like two opaque pieces of paper overlapping.<br/>- `multiply`<br/>  - : The final color is the result of multiplying the top and bottom colors.<br/>    A black layer leads to a black final layer, and a white layer leads to no change.<br/>    The effect is like two images printed on transparent film overlapping.<br/>- `screen`<br/>  - : The final color is the result of inverting the colors, multiplying them, and inverting that value.<br/>    A black layer leads to no change, and a white layer leads to a white final layer.<br/>    The effect is like two images shone onto a projection screen.<br/>- `overlay`<br/>  - : The final color is the result of `multiply` if the bottom color is darker, or `screen` if the bottom color is lighter.<br/>    This blend mode is equivalent to `hard-light` but with the layers swapped.<br/>- `darken`<br/>  - : The final color is composed of the darkest values of each color channel.<br/>- `lighten`<br/>  - : The final color is composed of the lightest values of each color channel.<br/>- `color-dodge`<br/>  - : The final color is the result of dividing the bottom color by the inverse of the top color.<br/>    A black foreground leads to no change. A foreground with the inverse color of the backdrop leads to a fully lit color.<br/>    This blend mode is similar to `screen`, but the foreground need only be as light as the inverse of the backdrop to create a fully lit color.<br/>- `color-burn`<br/>  - : The final color is the result of inverting the bottom color, dividing the value by the top color, and inverting that value.<br/>    A white foreground leads to no change. A foreground with the inverse color of the backdrop leads to a black final image.<br/>    This blend mode is similar to `multiply`, but the foreground need only be as dark as the inverse of the backdrop to make the final image black.<br/>- `hard-light`<br/>  - : The final color is the result of `multiply` if the top color is darker, or `screen` if the top color is lighter.<br/>    This blend mode is equivalent to `overlay` but with the layers swapped.<br/>    The effect is similar to shining a _harsh_ spotlight on the backdrop.<br/>- `soft-light`<br/>  - : The final color is similar to `hard-light`, but softer.<br/>    This blend mode behaves similar to `hard-light`.<br/>    The effect is similar to shining a _diffused_ spotlight on the backdrop*.*<br/>- `difference`<br/>  - : The final color is the result of subtracting the darker of the two colors from the lighter one.<br/>    A black layer has no effect, while a white layer inverts the other layer's color.<br/>- `exclusion`<br/>  - : The final color is similar to `difference`, but with less contrast.<br/>    As with `difference`, a black layer has no effect, while a white layer inverts the other layer's color.<br/>- `hue`<br/>  - : The final color has the _hue_ of the top color, while using the _saturation_ and _luminosity_ of the bottom color.<br/>- `saturation`<br/>  - : The final color has the _saturation_ of the top color, while using the _hue_ and _luminosity_ of the bottom color.<br/>    A pure gray backdrop, having no saturation, will have no effect.<br/>- `color`<br/>  - : The final color has the _hue_ and _saturation_ of the top color, while using the _luminosity_ of the bottom color.<br/>    The effect preserves gray levels and can be used to colorize the foreground.<br/>- `luminosity`<br/>  - : The final color has the _luminosity_ of the top color, while using the _hue_ and _saturation_ of the bottom color.<br/>    This blend mode is equivalent to `color`, but with the layers swapped.<br/><br/>## Description<br/><br/>For each pixel among the layers to which it is applied, a blend mode takes the colors of the foreground and the background, performs a calculation on them, and returns a new color value.<br/><br/>Changes between blend modes are not interpolated.
