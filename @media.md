## ✔️ @media 
 ---<br/>title: '@media'<br/>slug: Web/CSS/@media<br/>tags:<br/>  - '@media'<br/>  - At-rule<br/>  - CSS<br/>  - Conditional<br/>  - Reference<br/>browser-compat: css.at-rules.media<br/>---<br/>{{CSSRef}}<br/><br/>The **`@media`** [CSS](/en-US/docs/Web/CSS) [at-rule](/en-US/docs/Web/CSS/At-rule) can be used to apply part of a style sheet based on the result of one or more [media queries](/en-US/docs/Web/CSS/Media_Queries/Using_media_queries). With it, you specify a media query and a block of CSS to apply to the document if and only if the media query matches the device on which the content is being used.<br/><br/>> **Note:** In JavaScript, the rules created using `@media` can be accessed with the {{domxref(CSSMediaRule)}} CSS object model interface.<br/><br/>## Syntax<br/><br/>The `@media` at-rule may be placed at the top level of your code or nested inside any other [conditional group at-rule](/en-US/docs/Web/CSS/At-rule#conditional_group_rules).<br/><br/>```css<br/>/* At the top level of your code */<br/>@media screen and (min-width: 900px) {<br/>  article {<br/>    padding: 1rem 3rem;<br/>  }<br/>}<br/><br/>/* Nested within another conditional at-rule */<br/>@supports (display: flex) {<br/>  @media screen and (min-width: 900px) {<br/>    article {<br/>      display: flex;<br/>    }<br/>  }<br/>}<br/>```<br/><br/>For a discussion of media query syntax, please see [Using media queries](/en-US/docs/Web/CSS/Media_Queries/Using_media_queries#syntax).<br/><br/>## Description<br/><br/>### Media types<br/><br/>_Media types_ describe the general category of a device.<br/>Except when using the `not` or `only` logical operators, the media type is optional and the `all` type is implied.<br/><br/>- `all`<br/>  - : Suitable for all devices.<br/>- `print`<br/>  - : Intended for paged material and documents viewed on a screen in print preview mode. (Please see [paged media](/en-US/docs/Web/CSS/Paged_Media) for information about formatting issues that are specific to these formats.)<br/>- `screen`<br/>  - : Intended primarily for screens.<br/><br/>> **Note:** CSS2.1 and [Media Queries 3](https://drafts.csswg.org/mediaqueries-3/#background) defined several additional media types (`tty`, `tv`, `projection`, `handheld`, `braille`, `embossed`, and `aural`), but they were deprecated in [Media Queries 4](https://dev.w3.org/csswg/mediaqueries/#media-types) and shouldn't be used.<br/><br/>### Media features<br/><br/>_Media features_ describe specific characteristics of the {{glossary(user agent)}}, output device, or environment.<br/>Media feature expressions test for their presence or value, and are entirely optional. Each media feature expression must be surrounded by parentheses.<br/><br/>- {{cssxref(@media/any-hover, any-hover)}}<br/>  - : Does any available input mechanism allow the user to hover over elements?<br/>    Added in Media Queries Level 4.<br/>- {{cssxref(@media/any-pointer, any-pointer)}}<br/>  - : Is any available input mechanism a pointing device, and if so, how accurate is it?<br/>    Added in Media Queries Level 4.<br/>- {{cssxref(@media/aspect-ratio, aspect-ratio)}}<br/>  - : Width-to-height aspect ratio of the viewport<br/>- {{cssxref(@media/color, color)}}<br/>  - : Number of bits per color component of the output device, or zero if the device isn't color<br/>- {{cssxref(@media/color-gamut, color-gamut)}}<br/>  - : Approximate range of colors that are supported by the user agent and output device.<br/>    Added in Media Queries Level 4.<br/>- {{cssxref(@media/color-index, color-index)}}<br/>  - : Number of entries in the output device's color lookup table, or zero if the device does not use such a table<br/>- {{cssxref(@media/device-aspect-ratio, device-aspect-ratio)}}  {{deprecated_inline}}<br/>  - : Width-to-height aspect ratio of the output device.<br/>    Deprecated in Media Queries Level 4.<br/>- {{cssxref(@media/device-height, device-height)}} {{deprecated_inline}}<br/>  - : Height of the rendering surface of the output device.<br/>    Deprecated in Media Queries Level 4.<br/>- {{cssxref(@media/device-width, device-width)}} {{deprecated_inline}}<br/>  - : Width of the rendering surface of the output device. Deprecated in Media Queries Level 4.<br/>- {{cssxref(@media/display-mode, display-mode)}}<br/>  - : The display mode of the application, as specified in the web app manifest's [`display`](/en-US/docs/Web/Manifest#display) member.<br/>    Defined in the [Web App Manifest spec](https://w3c.github.io/manifest/#the-display-mode-media-feature).<br/>- {{cssxref(@media/dynamic-range, dynamic-range)}}<br/>  - : Combination of brightness, contrast ratio, and color depth that are supported by the user agent and the output device. Added in Media Queries Level 5.<br/>- {{cssxref(@media/forced-colors, forced-colors)}}<br/>  - : Detect whether user agent restricts color palette.<br/>    Added in Media Queries Level 5.<br/>- {{cssxref(@media/grid, grid)}}<br/>  - : Does the device use a grid or bitmap screen?<br/>- {{cssxref(@media/height, height)}}<br/>  - : Height of the viewport.<br/>- {{cssxref(@media/hover, hover)}}<br/>  - : Does the primary input mechanism allow the user to hover over elements?<br/>    Added in Media Queries Level 4.<br/>- {{cssxref(@media/inverted-colors, inverted-colors)}}<br/>  - : Is the user agent or underlying OS inverting colors?<br/>    Added in Media Queries Level 5.<br/>- {{cssxref(@media/monochrome, monochrome)}}<br/>  - : Bits per pixel in the output device's monochrome frame buffer, or zero if the device isn't monochrome.<br/>- {{cssxref(@media/orientation, orientation)}}<br/>  - : Orientation of the viewport.<br/>- {{cssxref(@media/overflow-block, overflow-block)}}<br/>  - : How does the output device handle content that overflows the viewport along the block axis?<br/>    Added in Media Queries Level 4.<br/>- {{cssxref(@media/overflow-inline, overflow-inline)}}<br/>  - : Can content that overflows the viewport along the inline axis be scrolled?<br/>    Added in Media Queries Level 4.<br/>- {{cssxref(@media/pointer, pointer)}}<br/>  - : Is the primary input mechanism a pointing device, and if so, how accurate is it?<br/>    Added in Media Queries Level 4.<br/>- {{cssxref(@media/prefers-color-scheme, prefers-color-scheme)}}<br/>  - : Detect if the user prefers a light or dark color scheme.<br/>    Added in Media Queries Level 5.<br/>- {{cssxref(@media/prefers-contrast, prefers-contrast)}}<br/>  - : Detects if the user has requested the system increase or decrease the amount of contrast between adjacent colors.<br/>    Added in Media Queries Level 5.<br/>- {{cssxref(@media/prefers-reduced-motion, prefers-reduced-motion)}}<br/>  - : The user prefers less motion on the page.<br/>    Added in Media Queries Level 5.<br/>- {{cssxref(@media/resolution, resolution)}}<br/>  - : Pixel density of the output device.<br/>- {{cssxref(@media/scripting, scripting)}}<br/>  - : Detects whether scripting (i.e. JavaScript) is available.<br/>    Added in Media Queries Level 5.<br/>- {{cssxref(@media/update-frequency, update)}}<br/>  - : How frequently the output device can modify the appearance of content.<br/>    Added in Media Queries Level 4.<br/>- {{cssxref(@media/video-dynamic-range, video-dynamic-range)}}<br/>  - : Combination of brightness, contrast ratio, and color depth that are supported by the video plane of user agent and the output device. Added in Media Queries Level 5.<br/>- {{cssxref(@media/width, width)}}<br/>  - : Width of the viewport including width of scrollbar.<br/><br/>### Logical operators<br/><br/>The _logical operators_ `not`, `and`, and `only` can be used to compose a complex media query.<br/>You can also combine multiple media queries into a single rule by separating them with commas.<br/><br/>- `and`<br/>  - : Used for combining multiple media features together into a single media query, requiring each chained feature to return `true` for the query to be `true`.<br/>    It is also used for joining media features with media types.<br/>- `not`<br/>  - : Used to negate a media query, returning `true` if the query would otherwise return `false`.<br/>    If present in a comma-separated list of queries, it will only negate the specific query to which it is applied.<br/>    If you use the `not` operator, you _must also_ specify a media type.<br/><br/>    > **Note:** In Level 3, the `not` keyword can't be used to negate an individual media feature expression, only an entire media query.<br/>- `only`<br/>  - : Applies a style only if an entire query matches.<br/>    It is useful for preventing older browsers from applying selected styles.<br/>    When not using `only`, older browsers would interpret the query `screen and (max-width: 500px)` as `screen`, ignoring the remainder of the query, and applying its styles on all screens.<br/>    If you use the `only` operator, you _must also_ specify a media type.<br/>- `,` (comma)<br/>  - : Commas are used to combine multiple media queries into a single rule.<br/>    Each query in a comma-separated list is treated separately from the others<br/>     Thus, if any of the queries in a list is `true`, the entire media statement returns `true`.<br/>     In other words, lists behave like a logical `or` operator.<br/><br/>## Accessibility concerns<br/><br/>To best accommodate people who adjust a site's text size, use [`em`](/en-US/docs/Learn/CSS/Building_blocks/Values_and_units#numeric_values)s when you need a {{cssxref(&lt;length&gt;)}} for your [media queries](/en-US/docs/Web/CSS/Media_Queries/Using_media_queries).<br/><br/>Both [`em`](/en-US/docs/Learn/CSS/Building_blocks/Values_and_units#numeric_values) and [`px`](/en-US/docs/Learn/CSS/Building_blocks/Values_and_units#numeric_values) are valid units, but [`em`](/en-US/docs/Learn/CSS/Building_blocks/Values_and_units#numeric_values) works better if the user changes the browser text size.<br/><br/>Also consider using Level 4 media queries to improve the user's experience. For example, `prefers-reduced-motion` to [detect if the user has requested that the system minimize the amount of animation](/en-US/docs/Web/CSS/@media/prefers-reduced-motion) or motion it uses.<br/><br/>## Security<br/><br/>Because media queries provide insights into the capabilities—and by extension, the features and design—of the device the user is working with, there is the potential that they could be abused to construct a fingerprint which identifies the device, or at least categorizes it to some degree of detail that may be undesirable to users.<br/><br/>Because of this potential, a browser may opt to fudge the returned values in some manner in order to prevent them from being used to precisely identify a computer. A browser might also offer additional measures in this area; for example, if Firefox's Resist Fingerprinting setting is enabled, many media queries report default values rather than values representing the actual device state.<br/><br/>## Formal syntax<br/><br/>{{csssyntax}}<br/><br/>## Examples<br/><br/>### Testing for print and screen media types<br/><br/>```css<br/>@media print {<br/>  body { font-size: 10pt; }<br/>}<br/><br/>@media screen {<br/>  body { font-size: 13px; }<br/>}<br/><br/>@media screen, print {<br/>  body { line-height: 1.2; }<br/>}<br/><br/>@media only screen<br/>  and (min-width: 320px)<br/>  and (max-width: 480px)<br/>  and (resolution: 150dpi) {<br/>    body { line-height: 1.4; }<br/>}<br/>```<br/><br/>Introduced in Media Queries Level 4 is a new range syntax that allows for less verbose media queries when testing for any feature accepting a range, as shown in the below examples:<br/><br/>```css<br/>@media (height > 600px) {<br/>    body { line-height: 1.4; }<br/>}<br/><br/>@media (400px <= width <= 700px) {<br/>    body { line-height: 1.4; }<br/>}<br/>```<br/><br/>For more examples, please see [Using media queries](/en-US/docs/Web/CSS/Media_Queries/Using_media_queries).<br/><br/>## Specifications<br/><br/>{{Specifications}}<br/><br/>## Browser compatibility<br/><br/>{{Compat}}<br/><br/>## See also<br/><br/>- [Using media queries](/en-US/docs/Web/CSS/Media_Queries/Using_media_queries)<br/>- In JavaScript, `@media` can be accessed via the CSS object model interface {{domxref(CSSMediaRule)}}.<br/>- [Extended Mozilla media features](/en-US/docs/Web/CSS/Mozilla_Extensions#media_features)<br/>- [Extended WebKit media features](/en-US/docs/Web/CSS/WebKit_Extensions#media_features)<br/>
