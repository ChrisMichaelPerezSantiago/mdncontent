## ✔️ text-transform 
 ---<br/>title: text-transform<br/>slug: Web/CSS/text-transform<br/>tags:<br/>  - CSS<br/>  - CSS Property<br/>  - Layout<br/>  - Reference<br/>  - Text<br/>  - recipe:css-property<br/>browser-compat: css.properties.text-transform<br/>---<br/>{{CSSRef}}<br/><br/>The **`text-transform`** CSS property specifies how to capitalize an element's text. It can be used to make text appear in all-uppercase or all-lowercase, or with each word capitalized. It also can help improve legibility for ruby.<br/><br/>{{EmbedInteractiveExample(pages/css/text-transform.html)}}<br/><br/>The `text-transform` property takes into account language-specific case mapping rules such as the following:<br/><br/>- In Turkic languages, like Turkish (`tr`), Azerbaijani (`az`), Crimean Tatar (`crh`), Volga Tatar (`tt`), and Bashkir (`ba`), there are two kinds of `i`, with and without the dot, and two case pairings: `i`/`İ` and `ı`/`I`.<br/>- In German (`de`), the `ß` becomes `SS` in uppercase.<br/>- In Dutch (`nl`), the `ij` digraph becomes `IJ`, even with `text-transform: capitalize`, which only puts the first letter of a word in uppercase.<br/>- In Greek (`el`), vowels lose their accent when the whole word is in uppercase (`ά`/`Α`), except for the disjunctive eta (`ή`/`Ή`). Also, diphthongs with an accent on the first vowel lose the accent and gain a diaeresis on the second vowel (`άι`/`ΑΪ`).<br/>- In Greek (`el`), the lowercase sigma character has two forms: `σ` and `ς`. `ς` is used only when sigma terminates a word. When applying `text-transform: lowercase` to an uppercase sigma (`Σ`), the browser needs to choose the right lowercase form based on context.<br/>- in Irish (`ga`), certain prefixed letters remain in lowercase when the base initial is capitalized, so for example `text-transform: uppercase` will change `ar aon tslí` to `AR AON tSLÍ` and not, as one might expect, `AR AON TSLÍ` (Firefox only). In some cases, a hyphen is also removed upon uppercasing: `an t-uisce` transforms to `AN tUISCE` (and the hyphen is correctly reinserted by `text-transform: lowercase`).<br/><br/>The language is defined by the [`lang`](/en-US/docs/Web/HTML/Global_attributes/lang) HTML attribute or the [`xml:lang`](/en-US/docs/Web/SVG/Attribute/xml:lang) XML attribute.<br/><br/>> **Note:** Support for language-specific cases varies between browsers, so check the [browser compatibility table](#browser_compatibility).<br/><br/>## Syntax<br/><br/>```css<br/>/* Keyword values */<br/>text-transform: none;<br/>text-transform: capitalize;<br/>text-transform: uppercase;<br/>text-transform: lowercase;<br/>text-transform: full-width;<br/>text-transform: full-size-kana;<br/><br/>/* Global values */<br/>text-transform: inherit;<br/>text-transform: initial;<br/>text-transform: revert;<br/>text-transform: revert-layer;<br/>text-transform: unset;<br/>```<br/><br/>- `capitalize`<br/><br/>  - : Is a keyword that converts the first _letter_ of each word to uppercase. Other characters remain unchanged (they retain their original case as written in the element's text). A letter is defined as a character that is part of Unicode's Letter or Number general categories {{experimental_inline}}; thus, any punctuation marks or symbols at the beginning of a word are ignored.<br/><br/>    > **Note:** Authors should not expect `capitalize` to follow language-specific title casing conventions (such as skipping articles in English).<br/><br/>    > **Note:** The `capitalize` keyword was under-specified in CSS 1 and CSS 2.1. This resulted in differences between browsers in the way the first letter was calculated (Firefox considered `-` and `_` as letters, but other browsers did not. Both Webkit and Gecko incorrectly considered letter-based symbols like `ⓐ` to be real letters. Internet Explorer 9 was the closest to the CSS 2 definition, but with some weird cases.) By precisely defining the correct behavior, CSS Text Level 3 cleans this mess up. The `capitalize` line in the browser compatibility table contains the version the different engines started to support this now precisely-defined behavior.<br/><br/>- `uppercase`<br/>  - : Is a keyword that converts all characters to uppercase.<br/>- `lowercase`<br/>  - : Is a keyword that converts all characters to lowercase.<br/>- `none`<br/>  - : Is a keyword that prevents the case of all characters from being changed.<br/>- `full-width`<br/>  - : Is a keyword that forces the writing of a character — mainly ideograms and Latin scripts — inside a square, allowing them to be aligned in the usual East Asian scripts (like Chinese or Japanese).<br/>- `full-size-kana`<br/>  - : Generally used for {{htmlelement(ruby)}} annotation text, the keyword converts all small Kana characters to the equivalent full-size Kana, to compensate for legibility issues at the small font sizes typically used in ruby.<br/><br/>## Accessibility concerns<br/><br/>Large sections of text set with a `text-transform` value of `uppercase` may be difficult for people with cognitive concerns such as Dyslexia to read.<br/><br/>- [MDN Understanding WCAG, Guideline 1.4 explanations](/en-US/docs/Web/Accessibility/Understanding_WCAG/Perceivable#Guideline_1.4_Make_it_easier_for_users_to_see_and_hear_content_including_separating_foreground_from_background)<br/>- [W3C Understanding WCAG 2.1](https://www.w3.org/TR/WCAG21/#visual-presentation)<br/><br/>## Formal definition<br/><br/>{{CSSInfo}}<br/><br/>## Formal syntax<br/><br/>{{csssyntax}}<br/><br/>## Examples<br/><br/>### Example using none<br/><br/>```html<br/><p>Initial String<br/>  <strong>Lorem ipsum dolor sit amet, consectetur adipisicing elit, ...</strong><br/></p><br/><p>text-transform: none<br/>  <strong><span>Lorem ipsum dolor sit amet, consectetur adipisicing elit, ...</span></strong><br/></p><br/>```<br/><br/>```css<br/>span {<br/>  text-transform: none;<br/>}<br/>strong { float: right;
