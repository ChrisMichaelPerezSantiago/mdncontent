## ✔️ clip-path 
 ---<br/>title: clip-path<br/>slug: Web/CSS/clip-path<br/>tags:<br/>  - CSS<br/>  - CSS Masking<br/>  - CSS Property<br/>  - Reference<br/>  - Web<br/>  - recipe:css-property<br/>browser-compat: css.properties.clip-path<br/>---<br/>{{CSSRef}}<br/><br/>The **`clip-path`** [CSS](/en-US/docs/Web/CSS) property creates a clipping region that sets what part of an element should be shown. Parts that are inside the region are shown, while those outside are hidden.<br/><br/>{{EmbedInteractiveExample(pages/css/clip-path.html)}}<br/><br/>## Syntax<br/><br/>```css<br/>/* Keyword values */<br/>clip-path: none;<br/><br/>/* <clip-source> values */<br/>clip-path: url(resources.svg#c1);<br/><br/>/* <geometry-box> values */<br/>clip-path: margin-box;<br/>clip-path: border-box;<br/>clip-path: padding-box;<br/>clip-path: content-box;<br/>clip-path: fill-box;<br/>clip-path: stroke-box;<br/>clip-path: view-box;<br/><br/>/* <basic-shape> values */<br/>clip-path: inset(100px 50px);<br/>clip-path: circle(50px at 0 100px);<br/>clip-path: ellipse(50px 60px at 0 10% 20%);<br/>clip-path: polygon(50% 0%, 100% 50%, 50% 100%, 0% 50%);<br/>clip-path: path('M0.5,1 C0.5,1,0,0.7,0,0.3 A0.25,0.25,1,1,1,0.5,0.3 A0.25,0.25,1,1,1,1,0.3 C1,0.7,0.5,1,0.5,1 Z');<br/><br/>/* Box and shape values combined */<br/>clip-path: padding-box circle(50px at 0 100px);<br/><br/>/* Global values */<br/>clip-path: inherit;<br/>clip-path: initial;<br/>clip-path: revert;<br/>clip-path: revert-layer;<br/>clip-path: unset;<br/>```<br/><br/>The `clip-path` property is specified as one or a combination of the values listed below.<br/><br/>### Values<br/><br/>- `<clip-source>`<br/>  - : A {{cssxref(url, url())}} referencing an [SVG](/en-US/docs/Web/SVG) {{SVGElement(clipPath)}} element.<br/>- {{cssxref(&lt;basic-shape&gt;)}}<br/><br/>  - : A shape whose size and position is defined by the `<geometry-box>` value. If no geometry box is specified, the `border-box` will be used as the reference box. One of:<br/><br/>    - {{cssxref(basic-shape/inset,inset())}}<br/>      - : Defines an inset rectangle.<br/>    - {{cssxref(basic-shape/circle,circle())}}<br/>      - : Defines a circle using a radius and a position.<br/>    - {{cssxref(basic-shape/ellipse,ellipse())}}<br/>      - : Defines an ellipse using two radii and a position.<br/>    - {{cssxref(basic-shape/polygon,polygon())}}<br/>      - : Defines a polygon using an SVG filling rule and a set of vertices.<br/>    - {{cssxref(path,path())}}<br/>      - : Defines a shape using an optional SVG filling rule and an SVG path definition.<br/><br/>- `<geometry-box>`<br/><br/>  - : If specified in combination with a `<basic-shape>`, this value defines the reference box for the basic shape. If specified by itself, it causes the edges of the specified box, including any corner shaping (such as a {{cssxref(border-radius)}}), to be the clipping path. The geometry box can be one of the following values:<br/><br/>    - `margin-box`<br/>      - : Uses the [margin box](/en-US/docs/Web/CSS/CSS_Shapes/From_box_values#margin-box) as the reference box.<br/>    - `border-box`<br/>      - : Uses the [border box](/en-US/docs/Web/CSS/CSS_Shapes/From_box_values#border-box) as the reference box.<br/>    - `padding-box`<br/>      - : Uses the [padding box](/en-US/docs/Web/CSS/CSS_Shapes/From_box_values#padding-box) as the reference box.<br/>    - `content-box`<br/>      - : Uses the [content box](/en-US/docs/Web/CSS/CSS_Shapes/From_box_values#content-box) as the reference box.<br/>    - `fill-box`<br/>      - : Uses the object bounding box as the reference box.<br/>    - `stroke-box`<br/>      - : Uses the stroke bounding box as the reference box.<br/>    - `view-box`<br/>      - : Uses the nearest SVG viewport as the reference box. If a {{SVGAttr(viewBox)}} attribute is specified for the element creating the SVG viewport, the reference box is positioned at the origin of the coordinate system established by the `viewBox` attribute and the dimension of the size of the reference box is set to the width and height values of the `viewBox` attribute.<br/><br/>- `none`<br/>  - : No clipping path is created.<br/><br/>> **Note:** A computed value other than **`none`** results in the creation of a new [stacking context](/en-US/docs/Web/CSS/CSS_Positioning/Understanding_z_index/The_stacking_context) the same way that CSS {{cssxref(opacity)}} does for values other than `1`.<br/><br/>## Formal definition<br/><br/>{{cssinfo}}<br/><br/>## Formal syntax<br/><br/>{{csssyntax}}<br/><br/>## Examples<br/><br/>###
